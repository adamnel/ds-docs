# 

Alignment

To prevent the empty state from jumping when switching to the progress or error state, define a container that is centered and align the empty state within it, ensuring that the layout remains consistent during state transitions.

# 

Button usage

Follow the rule of keeping only one primary button on each page when possible. If there is no primary button on the page, a primary button may be used in the empty state. If one exists already, use an outlined button or text button for less emphasis. 

# 

Content guidelines

Explain to the user that no data is available. Make it clear if this is an expected situation or an error. Provide a next step for the user to take to resolve the situation.

To pick up a draggable item, press the space bar. While dragging, use the arrow keys to move the item. Press space again to drop the item in its new position, or press escape to cancel.
