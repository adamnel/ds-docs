# Cards / Onboarding

Cards serve as shortcuts to tasks that we want people to complete first. It's an opportunity for us to help them quickly unlock the value of their Teradata feature or product.

## 

Anatomy

Onboarding cards

1.  Provides context and introduces the onboarding or task sequence using a title or message to succinctly communicate the overarching goal or status of the tasks.
    
2.  Wraps the onboarding cards in a collapsible format, which can be expanded or collapsed to show or hide the cards, helping manage screen real estate and focus.
    
3.  Visual indicator showing the user's progression through the onboarding steps, typically represented as "X/Y Complete," which helps motivate and guide users through the setup process.
    
4.  Positioned prominently to allow users to bypass the entire onboarding process or task list, enabling those familiar with the system to quickly move past introductory tasks.
    
5.  An icon button that changes state (e.g., up/down arrow) to control the expansion or collapse of the accordion container, indicating whether the accordion is open or closed.
    
6.  Positioned at the edges of the card carousel, these buttons allow users to view additional cards by navigating left or right through the onboarding card carousel.
    
7.  Individual containers for each task or step in the onboarding process, which include icons, titles, descriptions, and a call-to-action(CTA) button specific to each task, providing a clear and actionable path for users.
    
8.  Dots or other markers that indicate the current slide within the carousel and the total number of slides, with the currently active card highlighted in a darker shade to clearly show the user's position within the sequence.

## 

Content guidelines

![cv-cards_onboarding--do](https://design-src.knapsack.cloud/figma/covalent/KvXVAmYdCVAS7hzkOa73Em/2261921067357318529/images/15441-7306.png?Expires=1768219455&Key-Pair-Id=K3CKBA7ZQV44SC&Signature=GXnO29Oqybs9uYRAGfNCHhWWLwQyXTbW-rKO37GZ0M5roQmjCuQbVIcSlL39wqqdRf94MGwy9lRmjq9JpSD40IAEtjlSs~u0bEHX4exOg19C7h3FvxM-QZdSCX-1LRsZz62H5DDC69cCJZfImVrXP2eZ541YtOQOug9rQ24bsXZaeEYwQZYRHojBkrCKdS6MYldWmDp9tRQXxnEBuPVfvF-rGuQeaDxefsqo9SqXF4oOeuMxMzxKKM4N8GRWyiJ8SYgWCGpiw-KR2jue9wbMOeUjsrbFEfOQDkvYVrKQx8Y3NxZJRv5uCoIK3GdF2F1AOn7jeOWTQFKPRDf~ZvuImg__)

![cv-cards_onboarding--don't](https://design-src.knapsack.cloud/figma/covalent/KvXVAmYdCVAS7hzkOa73Em/2261921067357318529/images/15441-7326.png?Expires=1768219455&Key-Pair-Id=K3CKBA7ZQV44SC&Signature=zgIca4hBaqP7Xsc0j09oQcPzHsvxoAVAOo3YmF9qVF5hZMlPAjZHVVi63mtPgVJbeOtjkbwB~KFs1h6ARfh3FwsUYSIU2FLIrc1LU4n1so6KHqWqMJP0gflxUHmXcVaO0gGU1Rlwnn5D2W~d5JkSCgBPSLKxTgxzb9ZtrwWIct1cB0dxKCmgNORhowzttbaVVkyK-jBk-bSFbe5Y9WOAm7-~O3-tnK0ZXFJZZrk-qqXZZJzqow2uUbwwcT7LlLkEDi1oHGS-NJyCXssHz~yMy2n9YHVl7cIqAjImWe3j269F61u1aGoHUx7K1gA9zluggWFbxVtnGHvvcrPlY1j3lA__)

##### Do

**Heading**

Be as engaging as it can be.

Use imperative verbs and sentence style.

**Message**

When writing about benefits of completing the task, use the future sense to create a human-like tone, for example, _you'll_ or _they'll_.

If the task completion will enable functionalities for other people, be clear about who's doing what by using phrases like _they'll_. 

**Call to action**

Use strong verbs that correspond to the headings and encourage action. This ensures that the user can act confidently even if they skip reading the body of the message.

##### Don't

**Heading**

Don't sacrifice clarity. Content testing will tell you if a specific phrase isn't clear to the target audience.  

Don't use initial capitalization.

**Message**

Don't repeat the header. Instead, add details that will help the user succeed. 

Don't use the present sense when writing about benefits of completing the task. The future tense, for example, _They'll manage users...,_ is a better choice, because onboarding requires a conversational tone.

**Call to action**

Don't say "OK" or "Close." Design options to close or skip cards on the container.

##### Do

Plan for 30% text expansion to ensure there's room for readable and visually balanced content in the localized versions. With this in mind, follow these guidelines:

*   Optimize for the breakpoint width at 1280px because most Teradata users are on large screens.
    
*   On a standard 3-card layout, up to 2 lines in the English message is recommended.
    
*   On a 2-card layout, up to 1 line in the English message is recommended. 
    

##### Don't

Don't truncate any text.

Cards contain contextually rich information that users need to read in full to make decisions or take action. We use responsive designs so a card grows in height instead of truncating text when the user reduces their screen size. 

##### Do

Link to UIs where people can complete a task.

Onboarding cards are primarily meant for introducing workflows. They allow people to learn by doing.

##### Don't

Don't link to documentation.  

Use cards to link to UIs where people can complete a task, _not read about_ how to complete a task.

  

##### Do

Focus on the jobs to be done in the user's context. First, identify jobs that are most important for the user to unlock the value of the experience they're in. Then identify the role of the person who will perform the jobs. 

## 

States

### 

Action card states

The cards have 3 states:

1.  Incomplete - The task has not been completed.
    
2.  Completed - The task was successfully completed.
    
3.  Unavailable - The task is not available currently because is it dependent on an incomplete setup task.

### 

Scrollable

When the screen size is too small to display all cards at once, scroll controls should appear. The left and right arrow buttons can be used to scroll through the collection of cards. The carousel position is communicated by the dots beneath the cards.

Clicking the forward arrow on the last page of cards should return the user to the first page, and clicking the back arrow on the first page of cards should bring them to the last page.

The card carousel should be scrollable using the trackpad or arrow keys when the carousel is focused. Snap scrolling ensures that when users scroll through the carousel, the view always adjusts so that only fully visible cards are shown.
