# Checkbox / Usage

A checkbox allows the user to select one or more options from a set of choices. Unlike radio buttons, checkboxes allow for multiple selections, meaning the user can choose more than one option at a time.

# 

When to use

Checkboxes should be used when multiple selections are allowed from a set of options. If the user can only choose one option from a list, use **radio buttons** instead.

##### Do

Make sure each checkbox has a clear and concise label.

##### Do

*   Provide clear labels and group checkboxes logically. 
    
*   Use checkboxes for tasks like agreeing to terms and conditions, selecting multiple items, or confirming options. 
    
*   Ensure that checkboxes are large enough to be easily clickable and usable on all screen sizes. 
    
*   When necessary, include a "select all" checkbox that allows the user to select all available options. 
    
*   Use checkboxes when users need to select more than one option.
    

##### Don't

*   Use checkboxes for mutually exclusive choices—use **radio buttons** instead
    
*   **Overuse checkboxes for secondary or non-critical options.** This can overwhelm users and make forms feel cluttered.
    
*   Leave checkboxes unchecked without a clear indication of their current state.
    
*   **Rely on checkboxes alone for key actions.** Always provide clear labels and context to ensure users understand their purpose.
