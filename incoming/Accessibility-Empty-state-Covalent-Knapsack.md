# Empty state / Accessibility

Placeholder content displayed when there is no data to show.

# 

Overview

Empty states are designed to be accessible to everyone. They are built to ensure the UI state is understandable and actionable.

# 

Keyboard interactions

To support accessibility through keyboard navigation, empty states include the following interactions:

## 

Tab index focus

Ensures that the action buttons in empty states are included in sequential keyboard navigation, allowing users to reach and interact with them using the tab key.

## 

Hit area state

A visible focus indicator highlights the action buttons when they gain keyboard focus, ensuring users can easily identify which element is active.
