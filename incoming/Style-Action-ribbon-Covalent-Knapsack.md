# Action ribbon / Style

Provide a common, dynamic region for user actions and feedback which applies to a subset of the UI.

Action ribbons are split into two regions, with the section context (usually the title) aligned to the left, and the page actions aligned to the right.

1.  Icon to communicate status.
    
2.  Text to communicate status.
    
3.  Contains the action ribbon contents.
    
4.  Ribbon underline.
    
5.  All status related items.
    
6.  All actions, including icon buttons or regular buttons. Try to keep the number of page actions to three or less.

# 

Content Guidelines

Action Ribbons streamline workflows by facilitating user actions with clear icons and labels.

#### 

**Categorization**

**Primary Actions**: Use prominent buttons for essential actions.

**Secondary Actions**: Employ smaller icons for less frequent tasks.

#### 

**Voice & Tone:**

Direct and supportive, providing immediate feedback.

### 

  

#### 

**Label and Icon Use:**

**Labels:** Up to 3 words.

**Icons:** Universally recognizable.

**Tooltips:** Concise, no more than 30 characters.
