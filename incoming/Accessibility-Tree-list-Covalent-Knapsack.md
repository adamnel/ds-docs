# Tree list / Accessibility

Hierarchical list structure displaying data in a tree-like format.

# 

Overview

Tree lists require clear labeling, keyboard operability, and assistive technology compatibility to allow for accessible navigation through the hierarchy that is represented.

# 

Keyboard interactions

To support accessibility through keyboard navigation, tree lists include the following interactions:

*   **Tab:** Focus tree list items.
    
*   **Space or Enter:** Select the currently focused tree list item. Opens a currently closed item. Closes a currently opened item.
    
*   **Up/down arrow keys:** Move between tree list items.
    

## 

Tab index focus

Ensures that items in a tree list are included in the sequential keyboard navigation, allowing users to reach and interact with them using the Tab key.

  

### 

Hit area state

A visible focus indicator highlights items in the tree list when it gains keyboard focus, ensuring that users can easily identify which element is active.
