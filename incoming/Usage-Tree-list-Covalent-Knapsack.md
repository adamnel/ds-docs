# 

When to use

Tree lists provide a hierarchical drill-down of deep nested structures.

Variants

Purpose

[Default](https://design.teradata.com/site/covalent/3MEWkdMLax/pattern/tree-list?templateId=web-components-Ue0Hy3Asb&demoId=RjMe6Q4mA)

Provide access to hierarchical structures, such as databases or file structures, in an intuitive way.

# 

Recommendations

Recommended practices ensure clear guidance, usability, accessibility, appropriate sizing, and user support.

##### Do

*   **Do** use relevant icons as visual cues to help distinguish items in large lists.
    
*   **Do** keep labels as concise as possible when label content us not user generated, . 
    

##### Don't

*   **Don't** use tree lists to represent flat structures. Only use for hierarchical structures.
