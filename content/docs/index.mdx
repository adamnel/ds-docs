**Covalent** is the product design system for our applications. It unifies visual language, interaction patterns, and accessibility expectations so teams ship consistent, inclusive experiences without reinventing foundations on every project.

This site is organized into **Getting Started**, **Foundations**, **Components**, **Patterns**, **Tokens**, and **Content** (voice, tone, and writing guidance).

## How to use this site

Use the left navigation to move between sections. Each page includes practical examples, code snippets where appropriate, and notes on accessibility and content design. When you are unsure whether something is a **component** (reusable in code) or a **pattern** (how to combine components in a flow), start with **Foundations** or **Getting Started**, then jump to **Patterns** for end-to-end guidance.
