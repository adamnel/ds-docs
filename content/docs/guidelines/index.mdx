Guidelines capture **how we work** on Covalent: accessibility targets, documentation standards, and review expectations. They complement Foundations and Components—while those sections describe *what* to build, Guidelines describe *how we decide* something is ready for production.

## Audience

Contributors include designers, engineers, and technical writers. Docs should be understandable by **new hires** without a live walkthrough: define terms, link to patterns, and show realistic examples.

## Governance

Significant changes (new primitives, breaking component APIs) go through **design system council** review with a migration note. Smaller doc fixes can ship through the normal documentation PR process.
