Provide a common, dynamic region for user actions and feedback that applies to a **subset** of the UI (for example above a table or form region).

## Usage

<imagePlaceholder imageId="cv-action-ribbon_hero" />

### When to use

Use the action ribbon to surface **save**, **edit**, **delete**, **configure**, and related actions in a **consistent** place—especially in data-heavy or form-heavy views.

**Variants**

| Variant | Purpose |
| --- | --- |
| **Default** | Group related actions in a visible strip for complex tasks. |

### Bulk selection

When selection is enabled, the same strip can show **general** actions when nothing is selected and **bulk** actions when items are selected.

<imagePlaceholder imageId="cv-action-ribbon_bulk-selection_selected" />

### Mini-toolbar vs. action ribbon

These look similar but behave differently; they can be used together.

**Mini-toolbar**

- Contains a **title**
- Contains a **close** control
- Does **not** surface status feedback messages

<imagePlaceholder imageId="cv-action-ribbon_ribbon-vs-toolbar_toolbar" />

**Action ribbon**

- **No** title
- **No** close button
- **Does** show status and feedback messages

<imagePlaceholder imageId="cv-action-ribbon_ribbon-vs-toolbar_action-ribbon" />

### Recommendations

<doDont
  doBody={<>
    Use in **forms or data management** contexts where quick access helps. **Group** actions for the current page or region. **Prioritize** by frequency and keep placement **consistent** across the product.
  </>}
  dontBody={<>
    Use in **minimal** UIs with few actions, **overload** the strip with too many actions, or **mix unrelated** actions that confuse outcomes.
  </>}
/>

## Style

### Anatomy

<imagePlaceholder imageId="cv-action-ribbon_anatomy2" />

Action ribbons split into two regions: **context** (often a title or status on the left) and **actions** on the right.

1. Status icon
2. Status text
3. Ribbon content
4. Ribbon underline
5. Status-related items
6. Actions (buttons / icon buttons)—prefer **three or fewer** primary actions when possible.

### Typography

Use **caption** styles for ribbon text.

<imagePlaceholder imageId="cv-action-ribbon_typography" />

### Placement

Embed the ribbon **directly above** the region it affects.

<imagePlaceholder imageId="cv-action-ribbon_placement" />

### Action order

From **right to left**: primary add (if any), search, less common functions, overflow menu (if any).

<imagePlaceholder imageId="cv-action-ribbon_placement-order" />

### Area of influence

Actions and messages apply only to the region **immediately below** the ribbon.

<imagePlaceholder imageId="cv-action-ribbon_area-of-influence" />

### Content guidelines

Keep labels short (**up to three words**), icons **universal**, and tooltips **≤ ~30 characters**. Voice: direct and supportive.

## Accessibility

### Overview

The action ribbon exposes actions and status in one region; keyboard users must reach every control in a **logical order**, and disabled actions must be **skipped** in tab order.

### Keyboard

- **Tab** moves into the ribbon and through interactive controls.
- **Space** activates the focused action (buttons follow native behavior).

### Focus

Use **`tabindex="0"`** on interactive elements, clear **focus indicators**, and exclude **disabled** actions from the tab sequence.
