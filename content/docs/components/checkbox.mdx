A checkbox allows the user to select one or more options from a set of choices. Unlike radio buttons, checkboxes allow for multiple selections, meaning the user can choose more than one option at a time.

## Usage

<imagePlaceholder imageId="cv-checkbox--hero" />

### When to use

Checkboxes should be used when multiple selections are allowed from a set of options. If the user can only choose one option from a list, use **radio buttons** instead.

| Variant | Purpose |
| --- | --- |
| **Default** | Allow users to select multiple options independently. |

**Related component:** [Form field](/docs/components/form-field) -- A container allowing users to select both **checkbox** and **label**.

### Recommendations

<imagePlaceholder imageId="cv-checkbox_do--labels" />

<doDont
  doBody={<>
    Make sure each checkbox has a clear and concise label.

    * Provide clear labels and group checkboxes logically.
    * Use checkboxes for tasks like agreeing to terms and conditions, selecting multiple items, or confirming options.
    * Ensure that checkboxes are large enough to be easily clickable and usable on all screen sizes.
    * When necessary, include a "select all" checkbox that allows the user to select all available options.
    * Use checkboxes when users need to select more than one option.
  </>}
  dontBody={<>
    * Use checkboxes for mutually exclusive choices -- use **radio buttons** instead.
    * **Overuse checkboxes for secondary or non-critical options.** This can overwhelm users and make forms feel cluttered.
    * Leave checkboxes unchecked without a clear indication of their current state.
    * **Rely on checkboxes alone for key actions.** Always provide clear labels and context to ensure users understand their purpose.
  </>}
/>

### Anatomy

<imagePlaceholder imageId="cv-checkbox_anatomy" />

1. Contains the check icon.
2. Present if the checkbox is in the selected or indeterminate state.
3. Communicates the interaction state of the checkbox. States indicated by this layer include Hovered, Focused, and Pressed. See the Colors page under Foundations for more details on state layers.

### Content guidelines

The checkbox helps users make binary choices. Clear labeling and contextual guidance ensure correct usage and understanding.

* **Categorization**: Group related checkboxes with a clear group label.
* **Voice & tone**: Direct and supportive. Avoid jargon.
* **Word limit**: Limit label text to 3-5 words. Helper text can be up to 50 characters.

### Props

| Prop | Description |
| --- | --- |
| `checked` | Whether the checkbox is checked |
| `indeterminate` | Whether the checkbox is in the indeterminate state |
| `disabled` | Whether the checkbox is disabled |
| `reducedTouchTarget` | Whether to use a reduced touch target |
| `value` | The value of the checkbox |

### How to import

```js
import "@covalent/components/checkbox";
```

### Events

| Event name | Target | Detail | Description |
| --- | --- | --- | --- |
| `change` | `cv-checkbox` | `{}` | Fired when the user modifies the checkbox `checked` or `indeterminate` states from an input device interaction. Note that, like native `<input>`, the `change` event is *not* fired when the `checked` or `indeterminate` properties are set from JavaScript. |

## Accessibility

### Overview

Checkboxes support multiple selections and require clear labeling, focus states, and error handling to guide user actions effectively.

### Keyboard interactions

Checkboxes should be fully operable via keyboard:

* **Tab key**: Focuses the checkbox.
* **Spacebar**: Toggles the checkbox on or off.

<imagePlaceholder imageId="cv-checkbox_accessibility1" />

### Tab index focus

* **tabindex="0"** for active checkboxes.
* Provide clear focus and selection indicators.
* Exclude disabled checkboxes from the tab order.

<imagePlaceholder imageId="cv-checkbox_accessibility2" />
