Checkbox styled with an icon as its visual representation.

## Usage

<imagePlaceholder imageId="cv-checkbox-icon_hero" />

### When to use

Checkbox icons should be used when multiple selections are allowed from a set of options. These function identically to [standard checkboxes](/docs/components/checkbox), but present in a more visually impactful way, and provide a much larger click target.

### Recommendations

<doDont
  doBody={<>
    * Provide clear labels and group checkbox icons logically.
    * Use checkbox icons for tasks like agreeing to terms and conditions, selecting multiple items, or confirming options.
    * Ensure that checkbox icons are large enough to be easily clickable and usable on all screen sizes.
    * When necessary, include a "select all" checkbox icon that allows the user to select all available options.
    * Use checkbox icons when users need to select more than one option.
  </>}
  dontBody={<>
    * Use checkbox icons for mutually exclusive choices -- use [radio icons](/docs/components/radio-icon) instead.
    * **Overuse checkbox icons for secondary or non-critical options.** This can overwhelm users and make forms feel cluttered.
    * Leave checkbox icons unchecked without a clear indication of their current state.
  </>}
/>

### Content guidelines

The checkbox icon helps users make binary choices. Clear labeling and contextual guidance ensure correct usage and understanding.

* **Categorization**: Group related checkbox icons with a clear group label.
* **Voice & tone**: Direct and supportive. Avoid jargon.
* **Word limit**: Limit label text to 3-5 words. Helper text can be up to 50 characters.

### Props

| Prop | Description |
| --- | --- |
| `checked` | Whether the checkbox icon is checked |
| `disabled` | Whether the checkbox icon is disabled |
| `iconOnly` | Whether to show only the icon |
| `value` | The value of the checkbox icon |
| `width` | Width of the component |
| `height` | Height of the component |

### Events

| Event name | Target | Detail | Description |
| --- | --- | --- | --- |
| `change` | `cv-checkbox-icon` | `{}` | Fired when the user modifies the checkbox `checked` state from an input device interaction. Note that, like native `<input>`, the `change` event is *not* fired when the `checked` property is set from JavaScript. |

## Accessibility

### Overview

Checkboxes support multiple selections and require clear labeling, focus states, and error handling to guide user actions effectively.

### Keyboard interactions

Checkbox icons should be fully operable via keyboard:

* **Tab key**: Focuses the checkbox icon.
* **Spacebar**: Toggles the checkbox icon on or off.
* **Arrows**: Moves the selection.

<imagePlaceholder imageId="cv-checkbox-icon_accessibility1" />

### Tab index focus

* **tabindex="0"** for active checkboxes.
* Provide clear focus and selection indicators.
* Exclude disabled checkboxes from the tab order.
