Radio button styled with an icon as its visual representation.

## Usage

<imagePlaceholder imageId="cv-radio-icon_hero" />

### When to use

Use radio icon buttons when you need a more visual way for the user to choose one option from a set of predefined options, such as selecting a payment method or choosing a shipping speed. These function identically to standard radio buttons, but present in a more visually impactful way, and provide a much larger click target.

If there are more than two options, or the choices are less mutually exclusive, consider using checkboxes or checkbox icons.

### Recommendations

Recommended practices ensure clear guidance, usability, accessibility, appropriate sizing, and user support.

**Do:**

* Ensure radio icon buttons are clearly labeled with descriptive options.
* Ensure that the user cannot select multiple radio icon buttons in a set.
* Group radio icon buttons together.
* Provide a default selection if appropriate (e.g., defaulting to the most common choice).
* Make sure radio icon buttons are appropriately spaced to avoid accidental selection.

**Don't:**

* Make radio icon buttons too small to be easily clickable, especially on mobile.

### Content guidelines

Radio buttons allow users to select one option from a set. Clear labels and hierarchy reduce confusion.

* **Categorization**: Use for mutually exclusive options only.
* **Voice & tone**: Friendly and concise.
* **Word limit**: 2-4 words for button labels. Group label or helper text up to 50 characters.

### Props

| Prop | Description |
| --- | --- |
| `checked` | Whether the radio icon is checked |
| `disabled` | Whether the radio icon is disabled |
| `iconOnly` | Show icon only without text |
| `value` | Value of the radio icon |
| `name` | Name of the radio icon |
| `icon` | Material Symbol icon name |
| `text` | Text content for the radio icon |

### Code example

```html
<cv-radio-icon>
  <div slot="text">Balance Every week</div>
  <div slot="icon">
    <cv-icon> houseboat </cv-icon>
  </div>
</cv-radio-icon>
```

### Events

| Event name | Target | Detail | Description |
| --- | --- | --- | --- |
| `change` | `cv-radio-icon` | `{}` | Fired when the user modifies the radio `checked` state from an input device interaction on this radio. Note that, like native `<input>`, the `change` event is *not* fired when the `checked` property is set from JavaScript, nor is it fired when another radio in the same group becomes `checked`. |

## Accessibility

### Overview

Radio buttons enable mutually exclusive choices and must have clear labels and grouped accessibility support to ensure users can make informed decisions.

### Keyboard interactions

Radios should be fully operable via keyboard:

* **Tab key**: Moves focus to the radio icon group.
* **Spacebar**: Selects the currently focused radio icon.
* **Arrow keys**: Move focus between options and select one.

<imagePlaceholder imageId="cv-radio-icon_accessibility1" />

### Tab index focus

* Group the radio icons logically with a single **tabindex="0"** for the group.
* Focus states must indicate the selected item clearly.
* Exclude disabled options from the tab order.
