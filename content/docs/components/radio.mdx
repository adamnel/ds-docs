A radio button allows the user to select a single option from a set of mutually exclusive choices. It is typically used when only one option can be selected from a predefined set.

## Usage

<imagePlaceholder imageId="cv-radio_hero" />

### When to use

Use radio buttons when you need the user to choose one option from a set of predefined options, such as selecting a payment method or choosing a shipping speed. If there are more than two options, or the choices are less mutually exclusive, consider using checkboxes or a drop-down menu.

### Recommendations

Recommended practices ensure clear guidance, usability, accessibility, appropriate sizing, and user support.

<doDont
  doBody={<>
    Use radio buttons for mutually exclusive choices (e.g., "Male" / "Female," "Credit Card" / "PayPal").
  </>}
  dontBody={<>
    Use radio buttons for selections where the user needs to choose more than one option — use **checkboxes** for multiple selections.
  </>}
/>

<imagePlaceholder imageId="cv-radio_do-heading" />

**Do:** Label each radio button clearly and provide a grouping heading.

**Do:**

* Ensure radio buttons are clearly labeled with descriptive options.
* Ensure that the user cannot select multiple radio buttons in a set.
* Group radio buttons together.
* Provide a default selection if appropriate (e.g., defaulting to the most common choice).
* Make sure radio buttons are appropriately spaced to avoid accidental selection.

**Don't:**

* Make radio buttons too small to be easily clickable, especially on mobile.

### Content guidelines

Radio buttons allow users to select one option from a set. Clear labels and hierarchy reduce confusion.

* **Categorization**: Use for mutually exclusive options only.
* **Voice & tone**: Friendly and concise.
* **Word limit**: 2-4 words for button labels. Group label or helper text up to 50 characters.

## Accessibility

### Overview

Radio buttons enable mutually exclusive choices and must have clear labels and grouped accessibility support to ensure users can make informed decisions.

### Keyboard interactions

Radios should be fully operable via keyboard:

* **Tab key**: Moves focus to the radio group.
* **Spacebar**: Selects the currently focused radio option.
* **Arrow keys**: Move focus between options and select one.

<imagePlaceholder imageId="cv-radio_accessibility1" />

### Tab index focus

* Group the radio buttons logically with a single **tabindex="0"** for the group.
* Focus states must indicate the selected button clearly.
* Exclude disabled options from the tab order.
