## Web components

### Properties

| Property | Type | Description |
| --- | --- | --- |
| `rows` | `number` | Number of rows to display. |
| `cols` | `number` | Number of columns to display. |
| `value` | `string` | The input value. |
| `type` | `string` | Input type. One of: `text`, `search`, `tel`, `url`, `email`, `password`, `date`, `month`, `week`, `time`, `datetime-local`, `number`, `color`. |
| `label` | `string` | Label text displayed above the textarea. |
| `name` | `string` | Name of the input for form submission. |
| `placeholder` | `string` | Placeholder text when the field is empty. |
| `helper` | `string` | Helper text displayed below the textarea. |
| `helperPersistent` | `boolean` | Whether the helper text is always visible. |
| `outlined` | `boolean` | Whether to use the outlined variant. |
| `required` | `boolean` | Whether the field is required. |
| `disabled` | `boolean` | Whether the field is disabled. |
| `icon` | `string` | Leading icon to display. |
| `iconTrailing` | `string` | Trailing icon to display. |
| `maxLength` | `number` | Maximum character length. |
| `charCounter` | `boolean` | Whether to display a character counter. |
| `validationMessage` | `string` | Message to display when validation fails. |
| `validityState` | `object` | The validity state of the textarea. |
| `willValidate` | `boolean` | Whether the textarea will be validated. |
| `validityTransform` | `function` | Custom validity transform function. |
| `validateOnInitialRender` | `boolean` | Whether to validate on initial render. |

```html
<cv-textarea
  label="Text area label"
  outlined
  charCounter
  maxLength="80">
</cv-textarea>
```

### Methods

| Method | Description |
| --- | --- |
| `checkValidity() => boolean` | Returns `true` if the textarea passes validity checks. Returns `false` and fires an `invalid` event on the textarea otherwise. |
| `reportValidity() => boolean` | Runs `checkValidity()` method, and if it returns false, then it reports to the user that the input is invalid. |
| `setCustomValidity(message:string) => void` | Sets a custom validity message (also overwrites `validationMessage`). If this message is not the empty string, then the element is suffering from a custom validity error and does not validate. |
| `layout() => Promise<void>` | Re-calculate layout. If a textarea is styled with `display:none` before it is first rendered, and it has a label that is floating, then you must call `layout()` the first time you remove `display:none`, or else the notch surrounding the label will not render correctly. |
